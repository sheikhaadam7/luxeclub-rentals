Read the existing `app/actions/admin.ts` first. Preserve `verifyAdmin`, `updateVehicleAvailability`, and `getAdminDashboardData` exactly as they are. Add the following new Server Actions below the existing ones:

**Fleet Actions:**

1. `getFleetData()` — Returns all vehicles with their fields: `id, slug, name, category, daily_rate, weekly_rate, monthly_rate, is_available, is_active, override_notes, scraped_at, gps_device_id, deposit_amount`. Also returns the last scraper run and staleness flag (reuse logic from getAdminDashboardData). Also returns availability blocks per vehicle from `vehicle_availability_blocks`. Use admin client for full access.

2. `addVehicle(data)` — Accepts `{ name: string, slug: string, category: string, daily_rate: number, description?: string }`. Inserts a new vehicle with `is_available: true`, `is_active: true`. Validate slug format (lowercase, hyphens only). Return `{ error: string | null, vehicleId?: string }`.

3. `updateVehicle(vehicleId, data)` — Accepts partial vehicle fields: `{ name?, slug?, category?, daily_rate?, weekly_rate?, monthly_rate?, deposit_amount?, gps_device_id?, description?, override_notes? }`. Updates only provided fields. Uses admin client. Returns `{ error: string | null }`.

4. `toggleVehicleActive(vehicleId, isActive)` — Sets `is_active` on the vehicle. Returns `{ error: string | null }`.

5. `addAvailabilityBlock(vehicleId, startDate, endDate, reason?)` — Inserts into `vehicle_availability_blocks`. Validate that startDate <= endDate. Returns `{ error: string | null }`.

6. `removeAvailabilityBlock(blockId)` — Deletes from `vehicle_availability_blocks` by id. Returns `{ error: string | null }`.

**Booking Actions:**

7. `getAllBookings()` — Returns all bookings (not just one user's) with vehicle name and user email. Use admin client. Select: `id, vehicle_id, user_id, start_date, end_date, duration_type, status, payment_status, payment_method, total_due, pickup_method, delivery_address, created_at`. Join `vehicles(name, slug)` and `profiles(email:id)` — for profiles, query separately since RLS on profiles may limit joins. Order by `created_at DESC`.

8. `updateBookingStatus(bookingId, status)` — Accepts one of the 6 valid statuses. Updates the booking. This triggers Supabase Realtime automatically (bookings table is in publication). Add `updated_at: new Date().toISOString()`. Returns `{ error: string | null }`.

Every action must call `verifyAdmin()` first and return early with `{ error: 'Unauthorized' }` if not admin. Use `createAdminClient()` for all write operations. Add proper TypeScript types for all inputs and outputs.
