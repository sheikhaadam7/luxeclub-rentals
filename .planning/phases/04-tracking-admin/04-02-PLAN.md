**LiveTrackingMap (`components/tracking/LiveTrackingMap.tsx`)**

Create a `'use client'` component that renders a Mapbox GL JS v3 map with a live-updating car marker.

Props:
- `vehicleId: string` — for the Realtime location subscription
- `deliveryLat: number` — customer's delivery latitude
- `deliveryLng: number` — customer's delivery longitude
- `initialLat?: number` — last known car latitude (from vehicle_locations)
- `initialLng?: number` — last known car longitude

Implementation:
1. Import `mapbox-gl` and `'mapbox-gl/dist/mapbox-gl.css'` (CSS import is critical — without it the map renders broken).
2. Use `useRef<HTMLDivElement>` for the container and `useRef<mapboxgl.Map>` for the map instance (never useState for map).
3. Set `mapboxgl.accessToken = process.env.NEXT_PUBLIC_MAPBOX_TOKEN!`.
4. Initialize map with `style: 'mapbox://styles/mapbox/standard'`, centered on delivery location, zoom 13.
5. In `map.on('style.load', ...)` callback:
   - Set dark night theme: `map.setConfigProperty('basemap', 'lightPreset', 'night')` and `map.setConfigProperty('basemap', 'showPointOfInterestLabels', false)`.
   - Add a static delivery destination Marker with `color: '#0099ff'` (brand cyan) at deliveryLat/deliveryLng.
   - Add a GeoJSON source named `'car'` with a FeatureCollection containing the initial position (if provided) or empty features.
   - Add a circle layer `'car-marker'` on source `'car'` with paint: `circle-radius: 10`, `circle-color: '#ffffff'`, `circle-stroke-width: 2`, `circle-stroke-color: '#0099ff'`, `circle-emissive-strength: 1` (visible in night preset).
6. Use `useVehicleLocation({ vehicleId, initialLat, initialLng })` hook from `@/lib/hooks/use-vehicle-location`.
7. In a separate `useEffect` watching `lat`/`lng` from the hook: when coordinates change, call `(map.current?.getSource('car') as mapboxgl.GeoJSONSource)?.setData(...)` with the new coordinate as a Point feature. Also call `map.current?.panTo([lng, lat])` to follow the car.
8. Cleanup: `map.current?.remove()` in the useEffect return.
9. Render a `div` with `ref={mapContainer}`, `className="w-full rounded-[--radius-card] overflow-hidden"`, `style={{ height: '400px' }}`.
10. If `NEXT_PUBLIC_MAPBOX_TOKEN` is not set, render a fallback div with a message instead of the map (same pattern as Phase 3 address input).

**BookingStatusTimeline (`components/tracking/BookingStatusTimeline.tsx`)**

Create a `'use client'` component showing the booking status as a horizontal timeline with real-time updates.

Props:
- `bookingId: string`
- `initialStatus: string`

Implementation:
1. Use `useRealtimeBooking({ bookingId, initialStatus })` from `@/lib/hooks/use-realtime-booking`.
2. Define the 3 tracking steps as an array:
   - `{ key: 'confirmed', label: 'Rental Confirmed', icon: checkCircle }`
   - `{ key: 'car_on_the_way', label: 'Car On The Way', icon: truck }`
   - `{ key: 'car_delivered', label: 'Car Delivered', icon: mapPin }`
3. For each step, determine if it's `completed` (status has progressed past it), `active` (current status), or `upcoming`.
   - Order: confirmed < car_on_the_way < car_delivered
   - Completed steps: brand cyan color, solid line
   - Active step: brand cyan with pulse animation dot
   - Upcoming steps: brand-muted gray, dashed line
4. Render as a horizontal (desktop) / vertical (mobile) stepper with connecting lines between steps.
5. Use Tailwind classes matching the brand: `text-brand-cyan` for active, `text-brand-muted` for upcoming, `text-green-400` for completed.
6. Show a pulsing dot next to the active step label (e.g., `animate-pulse` on a small cyan dot).
7. For `cancelled` status, show a red cancelled banner instead of the timeline.
8. For `completed` status, show all 3 steps as completed with a "Rental Complete" message.
9. Use inline SVG icons (check circle, car/truck, map pin) — do NOT import an icon library.

Both components must use brand design tokens: `rounded-[--radius-card]`, `bg-brand-surface`, `border-brand-border`, `text-white`, `text-brand-muted`, `text-brand-cyan`.
