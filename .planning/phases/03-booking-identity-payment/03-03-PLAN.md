<objective>
Build the identity verification flow: a Server Action to create Veriff sessions, a booking wizard step that checks KYC status and either skips (if verified) or prompts verification, a Veriff widget/redirect component, a webhook handler that receives Veriff decisions and updates profiles.kyc_status, and the verify-callback return page.

Purpose: UAE rental regulations require passport and driving license verification before the first booking. This flow is triggered during the booking wizard (Step 4) only for unverified users, per the "deferred verification" decision. Users who have previously verified skip this step entirely.

Output: Veriff integration (session creation, redirect flow, webhook), StepIdentity component, identity Server Actions, verify-callback redirect page.
</objective>

  <action>
1. Create `app/actions/identity.ts` with 'use server' directive:

   **createVeriffSession():**
   - Get authenticated user via `createClient()` then `supabase.auth.getUser()` — return error if not authenticated
   - Check if user already has `kyc_status = 'verified'` on profiles — if so, return `{ alreadyVerified: true }`
   - POST to `https://stationapi.veriff.com/v1/sessions` with:
     - Headers: `X-AUTH-CLIENT: process.env.VERIFF_API_KEY!`, `Content-Type: application/json`
     - Body: `{ verification: { callback: process.env.NEXT_PUBLIC_APP_URL + '/book/verify-callback', person: { firstName: '', lastName: '' }, vendorData: user.id, timestamp: new Date().toISOString() } }`
   - Parse response: extract `verification.url` and `verification.id`
   - Update profiles with kyc_session_id and kyc_status='submitted' and kyc_provider='veriff'
   - Return `{ sessionUrl: string, sessionId: string }`

   **getVerificationStatus():**
   - Get authenticated user
   - Query profiles for `kyc_status` where `id = user.id`
   - Return `{ status: 'none' | 'submitted' | 'pending' | 'verified' | 'rejected' }`

2. Create `app/api/webhooks/veriff/route.ts`:
   - Export `POST(req: Request)` handler
   - Read raw body: `const body = await req.text()`
   - Verify Veriff webhook signature using HMAC-SHA256:
     - Signature is in `x-hmac-signature` header
     - Compute: `crypto.createHmac('sha256', process.env.VERIFF_SHARED_SECRET!).update(body).digest('hex')`
     - Compare with header value; return 401 if mismatch
   - Parse body as JSON
   - Handle event types:
     - `verification.approved` (action === 'approved'): set `kyc_status = 'verified'`, `kyc_verified_at = NOW()`
     - `verification.resubmission_requested`: set `kyc_status = 'pending'` (user needs to retry)
     - `verification.declined`: set `kyc_status = 'rejected'`
   - Extract `vendorData` from payload (this is the user ID set during session creation)
   - Use `createAdminClient()` to update profiles (bypasses RLS)
   - Return 200 OK
   - Import `createHmac` from `node:crypto`
  </action>

  </verify>
  <done>
