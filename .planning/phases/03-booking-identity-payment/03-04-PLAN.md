1. Create `app/actions/payments.ts` with 'use server' directive:

   **createRentalPaymentIntent(bookingId: string, amountAed: number):**
   - Import `stripe` from `@/lib/stripe/server`
   - Get authenticated user via createClient() -> getUser()
   - Create PaymentIntent: `stripe.paymentIntents.create({ amount: Math.round(amountAed * 100), currency: 'aed', capture_method: 'automatic', metadata: { booking_id: bookingId, user_id: user.id, type: 'rental' } })`
   - Update booking with stripe_rental_pi_id
   - Return `{ clientSecret: paymentIntent.client_secret }`

   **createDepositPaymentIntent(bookingId: string, depositAmountAed: number):**
   - Same auth check
   - Create PaymentIntent with `capture_method: 'manual'` and `payment_method_options: { card: { request_extended_authorization: 'if_available' } }` for >7 day holds
   - Metadata: `{ booking_id: bookingId, user_id: user.id, type: 'deposit' }`
   - Update booking with stripe_deposit_pi_id
   - Return `{ clientSecret: paymentIntent.client_secret }`

   **captureDeposit(bookingId: string):**
   - Admin-only (reuse verifyAdmin pattern from admin.ts)
   - Fetch booking's stripe_deposit_pi_id
   - Call `stripe.paymentIntents.capture(piId)`
   - Update booking deposit_status to 'captured'

   **voidDeposit(bookingId: string):**
   - Admin-only
   - Fetch booking's stripe_deposit_pi_id
   - Call `stripe.paymentIntents.cancel(piId)`
   - Update booking deposit_status to 'voided'

2. Create `app/actions/bookings.ts` with 'use server' directive:

   **createBooking(formData: BookingFormValues, vehicleId: string):**
   - Get authenticated user
   - Fetch vehicle from DB to get current rates (NEVER trust client-side rates)
   - Server-side price recalculation: call `calculateBookingTotal(vehicle, formData)` from `@/lib/pricing/calculator`
   - Insert booking row into Supabase with ALL fields:
     - vehicle_id, user_id, start_date, end_date, duration_type, start_time
     - pickup_method, delivery_address, delivery_lat, delivery_lng, return_method
     - deposit_choice, deposit_amount, rental_subtotal, delivery_fee, return_fee, no_deposit_surcharge, total_due
     - payment_method (from formData.paymentMethod)
     - status: 'pending', payment_status: formData.paymentMethod === 'cash' ? 'pending_cash' : 'unpaid'
   - Return booking ID and price breakdown
   - If paymentMethod is 'cash': booking is created with status 'pending' and payment_status 'pending_cash' — no Stripe involvement
   - If paymentMethod is card/apple_pay/google_pay:
     - Call createRentalPaymentIntent with the server-calculated totalDue
     - If deposit chosen: also call createDepositPaymentIntent with deposit amount
     - Return both clientSecrets to the client

3. Create `app/api/webhooks/stripe/route.ts`:
   - Export `POST(req: Request)` handler
   - CRITICAL: read raw body with `await req.text()` (NOT req.json())
   - Get signature from `(await headers()).get('stripe-signature')`
   - Verify with `stripe.webhooks.constructEvent(body, sig, process.env.STRIPE_WEBHOOK_SECRET!)`
   - Catch verification error -> return 400
   - Idempotency: insert into `stripe_webhook_events` with event.id. If unique constraint error (code 23505), return 200 (already processed)
   - Use `createAdminClient()` for all DB operations (webhook has no user session)
   - Handle event types:
     - `payment_intent.succeeded`: if metadata.type === 'rental', update booking payment_status to 'paid'. If metadata.type === 'deposit', this means the deposit was charged (shouldn't happen with manual capture unless explicitly captured).
     - `payment_intent.requires_capture`: metadata.type === 'deposit' — update booking deposit_status to 'held'
     - `payment_intent.canceled`: update booking deposit_status to 'voided'
     - `payment_intent.payment_failed`: update booking payment_status to 'failed'
   - Return 200 OK
   - Import `stripe` from `@/lib/stripe/server`
   - Import `headers` from `next/headers`
  </action>
