Update `app/globals.css` to add the full `@theme` design token block. These tokens auto-generate Tailwind utility classes like `bg-brand-black`, `text-brand-cyan`, `font-display`, etc. The `bg-luxury` utility class is a custom CSS class added alongside `@theme`:

```css
@import "tailwindcss";

@theme {
  /* Brand Colors */
  --color-brand-black: #000000;
  --color-brand-surface: #0a0a0a;
  --color-brand-border: rgba(255, 255, 255, 0.15);
  --color-brand-cyan: #0099ff;
  --color-brand-cyan-hover: #0077cc;
  --color-brand-text: #ffffff;
  --color-brand-muted: rgba(255, 255, 255, 0.6);
  --color-brand-subtle: rgba(255, 255, 255, 0.15);

  /* Typography — CSS variable names must match what next/font assigns in layout.tsx */
  --font-display: var(--font-display), Georgia, serif;
  --font-body: var(--font-body), system-ui, sans-serif;

  /* Radii */
  --radius-card: 0.75rem;
}

/* Luxury radial gradient background — used on the landing page */
.bg-luxury {
  background: radial-gradient(
    ellipse at 50% 0%,
    #1a1a1a 0%,
    #000000 65%
  );
}

/* Subtle border glow on focus for auth inputs */
.input-focus-glow:focus {
  box-shadow: 0 0 0 1px #0099ff40;
}
```

Note on fonts: `next/font/google` sets CSS variables `--font-display` and `--font-body` on the `<html>` element. Tailwind v4 `@theme` references these via `var(--font-display)` — they chain correctly at runtime because both use the same CSS variable names.
